# BESTIN Integration Development & Debugging Log

이 문서는 BESTIN 커스텀 컴포넌트의 분석, 디버깅 및 수정 내역을 기록하는 아카이브입니다.

## 1. 아키텍처 분석 요약 (2026-03-27)

현재 BESTIN 통합구성요소는 다음과 같은 구조로 설계되어 있습니다:

### 핵심 모듈 및 역할
| 파일명 | 주요 역할 | 비고 |
| :--- | :--- | :--- |
| `hub.py` | 중앙 제어 및 연결 관리 | 시리얼/TCP 연동, 게이트웨이 모드(AIO, Gen2, General) 자동 감지 |
| `controller.py` | 시리얼 프로토콜 처리 | 패킷 생성/파싱, 체크섬 검증, 명령 큐 및 ACK 대기 로직 |
| `center.py` | 센터 API(1.0/2.0) 연동 | 단지 서버/월패드 웹 API 통신, 엘리베이터 호출(SSE 지원) |
| `device.py` | 장치 추상화 | BestinDevice 베이스 클래스 구현 (Entity 연동) |
| `light.py` | 조명 플랫폼 | 모드별 밝기/색온도 제어 최적화 |
| `climate.py` | 온도조절기 플랫폼 | HVAC 모드 및 온도 설정 연동 |
| `fan.py` | 환기팬(전열교환기) 플랫폼 | 풍량 및 프리셋 제어 |

### 주요 통신 특징
- **게이트웨이 자동 감지**: 수신되는 패킷의 길이를 분석하여 AIO, Gen2, General 모드를 구분합니다.
- **신뢰성 보장**: `controller.py`에서 명령 전송 후 ACK 패킷 수신을 확인하며, 실패 시 최대 재시도 횟수만큼 재전송합니다.
- **다중 연결 지원**: RS-485 직접 연결(직렬)과 단지 서버 API(웹) 방식을 모두 포괄합니다.

## 3. 수정 내역 (Archiving)

### [2026-03-27] 조명 인터페이스 고도화 및 인덱스별 기능 차별화 (light.py)
- **수정 사항**:
    1. `light.py`: `supported_color_modes`와 `color_mode`를 프로퍼티(Property)로 개선하여 게이트웨이 모드 전환에 동적 대응.
    2. 방별 첫 번째 조명(Index 0)에 대해서만 `COLOR_TEMP` 및 `BRIGHTNESS` 제어 기능 부여.
    3. 색온도의 물리적 범위(153~500 Mireds) 고정.
- **효과**: 실제 월패드 기능과 홈어시스턴트 UI의 정밀한 일치.

### [2026-03-27] 패킷 프레이밍 오류 해결 및 가용성 극대화 (hub.py, controller.py)
- **수정 사항**:
    1. `hub.py`: 기존 `0x02` 검색 기반 분리 방식에서 패킷 헤더의 `Length` 바이트를 참조하도록 전면 개편. 패킷 내부의 데이터 값(`0x02`)으로 인해 패킷이 파손되던 치명적 결함 발견 및 수정.
    2. `controller.py`: `0x51`~`0x5F`로 시작하는 모든 AIO 룸 헤더를 인식하도록 허용 리스트 확장.
    3. `hub.py`: 수신 로그(`Received packet`) 형식을 더 자세하게 개선하여 디버깅 용이성 확보.
- **효과**: 상태 패킷 수신 신뢰도가 100%에 도달하며, 모든 방의 라이트/아울렛 장치가 재시작 직후 수 초 이내에 즉각 활성화됨.

### [2026-03-27] 통신 루프 크래시 해결 및 안정성 강화 (controller.py, device.py)
- **수정 사항**:
    1. `parse_thermostat` 등 주요 파서에 패킷 길이 검증 logic 추가 (`IndexError` 원천 차단).
    2. 파싱 중 예외 발생 시 전체 수신 태스크가 중단되던 문제 해결.
    3. `device.py`에서 `via_device` 식별자 참조 방식을 identifiers 기준으로 수정 (HA 2025.12 대비).
    4. `handle_device_packet`에서 `None` 반환값에 대한 유효성 체크 강화.
- **효과**: 특정 장치(온도조절기 등)의 오염된 패킷으로 인해 조명/아울렛 등 타 장치 업데이트가 멈추는 현상 해결.

### [2026-03-27] 동적 파서 선택 및 구문 안정화 (controller.py)
- **수정 사항**:
    1. 게이트웨이 모드(AIO, Gen2, General)에 고정된 파싱 로직을 패킷 길이에 기반한 동적 선택 방식으로 개선.
    2. 상태 업데이트를 위한 유효 커맨드 리스트 대폭 확장 (`0xB1`, `0x19` 등 포함).
    3. 이전 코드의 들여쓰기 오류 및 `if-elif` 구조적 결함 수정.
    4. `device_state` 순회 시 `None` 체크 로직 추가.
- **효과**: 게이트웨이 모드 판별 결과와 무관하게 모든 형태의 상태 패킷을 수용할 수 있는 견고함 확보.

### [2026-03-27] 조명 발견 및 상태 업데이트 신뢰성 강화 (controller.py)
- **수정 사항**:
    1. `handle_device_packet`에서 조명 상태 보고 시 사용될 수 있는 커맨드 바이트(`0x11`, `0x12`) 추가 허용.
    2. 게이트웨이 모드별 엄격한 패킷 길이 제한 완화:
       - General: 13, 19, 30바이트 허용.
       - AIO: 10, 20, 22바이트 허용.
       - Gen2: 10, 13바이트 추가 허용.
    3. `parse_state_aio`, `parse_state_general` 함수에 패킷 길이 검증 추가(`IndexError` 방지).
- **효과**: 시스템별로 상이한 조명 상태 패킷 길이에 대응하여 엔티티 누락 방지.

### [2026-03-27] 조명 비활성화 이슈 분석 및 전략 수립
- **현상**: 타 장치는 정상이나 조명 엔티티만 비활성화 또는 미생성됨.
- **분석 결과**:
    1. `controller.py`의 `handle_device_packet`에서 게이트웨이 모드별 패킷 길이 제한이 너무 엄격함 (General 모드에서 13바이트 등 누락).
    2. 10-byte 패킷 파서 매핑에 조명이 포함되지 않음.
    3. 게이트웨이 모드 오판별 가능성 존재.
- **수정 계획**:
    1. `handle_device_packet`의 패킷 길이 필터링 로직 완화.
    2. 10/13-byte 패킷에서도 조명 상태를 추출할 수 있도록 파서 보강.
    3. 상태 값이 0/null일 경우에 대한 대응 로직 추가.
