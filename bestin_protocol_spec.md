# BESTIN Protocol Specification (Code Analysis)

이 문서는 현재 `controller.py`에 구현된 장치별 제어(Send) 및 상태(Receive) 패킷의 구조를 정리한 것입니다.

## 1. 공통 패킷 구조 (Common Packet)
- **Structure**: `[0x02, Header, Length, PacketType, Timestamp, Data..., Checksum]`
- **Header**: `0x28`(온도), `0x31`(조명/아울렛/가스), `0x41`(도어락), `0x61`(팬), `0xD1`(에너지) 등
- **Checksum**: XOR 연산 기반 (`calculate_checksum` 함수 참조)

---

## 2. 장치별 제어 패킷 (HA -> BESTIN)

### 조명 (Light) 💡
| 게이트웨이 모드 | 헤더 | 길이 | 타입 | 데이터 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **AIO** | `0x50+RoomID` | `0x0A` | `0x12` | `Data[5]`: ON/OFF, `Data[6]`: 위치 비트, `Data[7/8]`: 색온도/밝기 |
| **Gen2** | `0x30+RoomID` | `0x0E` | `0x21` | `Data[8, 9, 10]`: ON/OFF, 밝기, 색온도 |
| **General** | `0x31` | `0x0D` | `0x01` | `Data[5]`: RoomID, `Data[6]`: 위치 및 ON/OFF 비트 |

### 콘센트 (Outlet/Switch) 🔌
| 게이트웨이 모드 | 헤더 | 길이 | 타입 | 데이터 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **AIO** | `0x50+RoomID` | `0x0C` | `0x12` | `Data[10]`: ON/OFF |
| **Gen2** | `0x30+RoomID` | `0x09` | `0x22` | `Data[5, 6, 7]`: 고정값, 위치, ON/OFF |
| **General** | `0x31` | `0x0D` | `0x01` | `Data[5, 7, 11]`: RoomID, 위치 비트, ON/OFF |

### 기타 장치
- **온도조절기**: 헤더 `0x28`, 길이 `0x0E`, 타입 `0x12`. 주방/거실 등 RoomID에 따라 데이터 분기.
- **가스/도어락/팬**: 헤더 `0x31/0x41/0x61`, 길이 `0x0A`. 기본 10바이트 명령 구조.

---

## 3. 상태 수신 패킷 (BESTIN -> HA)

현재 코드는 패킷 길이를 기반으로 파서를 동적으로 선택합니다.

### 상태 파싱 로직 (Discovery)
1. **General (30바이트)**: 헤더 `0x31` 등. `packet[6]`에서 조명 상태, `packet[7]`에서 아울렛 상태 추출.
2. **AIO (20/22바이트)**: 헤더 `0x51`, `0x52` 등. `packet[6]` 비트 연산으로 조명 상태 확인.
3. **Gen2 (Variable)**: `27, 41, 55, 59, 72, 98, 150`바이트 등. 인덱스를 계산하여 조명/아울렛 데이터 추출.
4. **10바이트 장치**: 가스(`0x31`), 도어락(`0x41`), 팬(`0x61`)은 고정 10바이트 패킷으로 즉각 파싱.

---

## 4. 디버깅 참고 사항
- **통신 누락 원인**: 사용자의 실제 시스템 패킷 길이가 위 정의된 리스트에 없을 경우 데이터가 무시됨.
- **체크섬 오류**: 패킷 끝에 있는 1바이트 체크섬이 틀리면 파싱되지 않음 (Packet Viewer를 통해 확인 권장).
